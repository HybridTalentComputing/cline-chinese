## 你将获得什么

它提供平台团队在生产部署中所需的五个核心功能。每个功能都解决了在整个组织中扩展 AI 编程的特定要求。

### 自带推理

使用你现有的云合同和协商的费率。大多数 AI 工具强制你通过它们购买带有加价的推理。Cline 直接连接到你的提供商。

连接到任何推理提供商：
- AWS Bedrock
- Google Vertex AI
- Azure OpenAI
- Anthropic direct
- OpenAI direct
- Cerebras
- 任何 OpenAI 兼容的端点

当新模型发布时立即切换模型。使用 Claude Sonnet 4.5 作为日常驱动，GPT-5 用于复杂重构，开源模型用于简单任务。你现有的云信用和初创项目合同现在涵盖 AI 编程。我们处理代理循环。你处理推理。没有加价，没有供应商锁定。

### 规模化治理

当数千名开发者使用 AI 时，平台团队需要集中控制。分散在笔记本电脑上的个人 API 密钥会造成安全风险和成本超支。

企业治理提供：
- **SSO 身份验证**：企业凭据代替个人 API 密钥
- **基于角色的访问控制**：三层层次结构（Member/Admin/Owner），具有组织范围的权限
- **模型和工具控制**：管理每个团队访问的模型和工具
- **远程配置**：从一个仪表板为所有开发者管理设置
- **使用跟踪和可观察性**：OpenTelemetry 集成，用于监控使用情况、成本和性能，并为管理操作提供选择性审计日志

一次配置，随处部署。开发者按自己的偏好工作，而你保持控制。

### 完整的可观察性

将日志导出到你现有的可观察性堆栈。跟踪所有团队的使用情况、成本和性能。

- **OpenTelemetry 导出**：与 Datadog、Grafana、Splunk 直接集成
- **实时分析**：跟踪采用情况、性能和模式
- **成本细分**：准确查看每个团队在哪些模型上花费了什么
- **JSON 输出**：在你现有的工具中构建自定义仪表板

与生产系统所需的可观察性标准相同。

## 部署

Cline 企业版安全连接到你的基础设施。在云环境中部署。配置以与你现有的安全策略和合规要求一起工作。

部署到你的组织：
1. 配置 Cline Core 以连接到你的基础设施
2. 设置 SSO、RBAC 和治理策略
3. 通过你现有的软件分发部署到开发者
4. 通过你的可观察性工具监控使用情况
