### 概述

为了帮助让 Cline 对每个人都更好，我们收集使用数据，帮助我们了解开发者如何使用我们的开源 AI 编程代理。这个反馈循环对于改进 Cline 的功能和用户体验至关重要。

我们使用 PostHog，一个开源分析平台，进行数据收集和分析。我们的数据采集实现完全透明 - 你可以查看[源代码](https://github.com/cline/cline/blob/main/src/services/posthog/telemetry/TelemetryService.ts)来确切了解我们跟踪的内容。

### 跟踪策略

隐私是我们的首要任务。默认情况下，所有收集的数据都是匿名的。如果你使用 Cline 账户登录，你的数据采集数据将与你的账户关联，以帮助我们在你遇到问题时改进产品并提供更好的支持。你的代码、提示和对话内容始终保持私密，永远不会被收集。

### 我们跟踪的内容

我们收集基本的使用数据，包括：

**任务交互：**任务开始和完成时、对话流程（不含内容）\
**模式和工具使用：**在计划/执行模式之间切换、正在使用哪些工具\
**Token 使用：**关于对话长度的基本指标以估算成本（不是 token 的实际内容）\
**系统上下文：**操作系统类型和 VS Code 环境详细信息\
**UI 活动：**导航模式和功能使用

为了完全透明，你可以检查我们的[数据采集实现](https://github.com/cline/cline/blob/main/src/services/posthog/telemetry/TelemetryService.ts)来查看我们跟踪的确切事件。

### 如何选择退出

Cline 中的数据采集是完全可选的：

-   当你更新或安装我们的 VS Code 扩展时，你会看到关于我们数据采集的消息
-   你可以随时在设置中更改你的首选项

Cline 也尊重 VS Code 的全局数据采集设置。如果你在 VS Code 级别禁用了数据采集，Cline 的数据采集也将自动禁用。
