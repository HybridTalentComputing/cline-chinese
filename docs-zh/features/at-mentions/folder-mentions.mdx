文件夹引用让你可以将整个目录带入与 Cline 的对话中。只需输入 `@/` 后跟一个以斜杠结尾的文件夹路径，Cline 就可以访问文件夹结构及其内容。

当你在聊天中输入 `@/` 时，Cline 显示你的工作区文件和文件夹。导航到你想要的文件夹，确保包含尾部斜杠，Cline 将看到文件夹的结构和内容。

当我需要理解或重构我的代码库的整个部分时，我使用文件夹引用。与其逐个引用单独的文件，我可以直接指向整个目录：

```
我试图理解我的应用程序中的身份验证流是如何工作的。
你能解释 @/src/auth/ 中文件的结构和关系吗？
```

然后 Cline 可以看到 auth 目录中的所有文件、它们的内容以及它们之间的关系。这给了它完整的上下文来解释多个文件之间的复杂交互。

文件夹引用也非常适合获得项目组织的帮助。当我不确定我的项目结构是否有意义时，我会让 Cline 审查它：

```
我正在设置一个新的 React 项目。这个文件夹结构有意义吗？@/src/
你会更改什么使其在项目增长时更可维护？
```

下次你处理多个相关文件时，尝试使用文件夹引用而不是单独引用每个文件。你会获得更全面的帮助，因为 Cline 可以看到一切如何组合在一起的更大画面。

## 底层工作原理

当你在消息中使用文件夹引用时，幕后发生的事情如下：

1. 当你发送消息时，Cline 检测文本中的 `@/path/to/folder/` 模式（带有尾部斜杠）
2. 扩展解析相对于你工作区根目录的文件夹路径
3. 它调用 `fs.readdir()` 获取该文件夹中所有文件和子目录的列表
4. 对于目录中的每个文件，它检查它是二进制还是基于文本的
5. 对于文本文件，它提取完整内容
6. 文件夹结构和文件内容以结构化格式附加到你的消息中：

    ```
    <folder_content path="path/to/folder">
    ├── file1.txt
    ├── file2.js    └── subfolder/

    <file_content path="path/to/folder/file1.txt">
    [文件内容]
    </file_content>

    <file_content path="path/to/folder/file2.js">
    [文件内容]
    </file_content>
    </folder_content>
    ```

7. 此增强消息及其嵌入的文件夹结构和文件内容被发送到 AI
8. AI 现在"看到"目录结构和该目录中文件的内容

每当你使用文件夹引用时，这个过程自动发生，向 AI 提供你的项目结构和文件内容的全面视图。
