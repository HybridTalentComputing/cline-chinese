当你遇到 Cline 的奇怪问题时，`/reportbug` 绝对是救命稻草。不必记住 GitHub 在错误报告中想要的所有细节，此命令将 Cline 变成你的个人错误报告助手。

它引导你收集正确错误报告所需的所有信息，然后将其直接发送到我们的 GitHub issues 页面，包括所有正确的格式和系统详细信息。

#### 使用 `/reportbug` 斜杠命令

当你遇到看起来不对劲的奇怪问题时：

-   只需在聊天中输入 `/reportbug`
-   Cline 将引导你完成我们需要的所有细节：
    -   描述问题的快速标题
    -   实际发生的事情与你期望的事情
    -   重现错误的步骤
    -   你看到的任何相关输出或错误
    -   可能帮助我们修复它的任何其他上下文
    -   你将在提交之前审查所有内容
    -   一旦你批准，它就会打开一个格式完美的 GitHub issue，包含你的所有信息以及自动系统详细信息

#### 示例

上周我遇到了一个奇怪的错误，Cline 在读取大文件时一直超时。我不必试图记住所有 GitHub 模板字段，只是输入了 `/reportbug`，Cline 引导我完成了整个过程。

它询问了我试图做什么，实际发生了什么，以及导致问题的确切步骤。最好的部分是它自动包含了我的操作系统版本、Cline 版本，以及我们的开发人员需要的所有技术细节。

几秒钟后，我创建了一个格式正确的 GitHub issue，而不必自己查找任何信息。
