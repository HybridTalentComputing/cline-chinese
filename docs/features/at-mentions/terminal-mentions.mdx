终端提及让您可以将终端输出直接引入与 Cline 的对话中。只需输入 `@terminal`，Cline 就能看到您终端的最近输出。

当您在聊天中输入 `@` 时，从菜单中选择"终端"或直接输入 `@terminal`。Cline 将立即看到您活动终端的最近输出，包括错误消息、构建日志或命令结果。

当我在处理构建错误、测试失败或调试输出时，我会一直使用终端提及。无需尝试复制和粘贴终端输出（这通常会丢失格式），我只需询问：

```
运行测试时遇到这个错误：@terminal

是什么原因导致的，我如何修复它？
```

这给 Cline 提供了格式完整的完整终端输出。然后 Cline 可以分析错误消息、堆栈跟踪和周围上下文，以提供更准确的帮助。

当与文件提及结合使用时，终端提及特别强大。当我调试失败的 API 调用时，我会引用两者：

```
调用我的 API 时遇到这个错误：@terminal

这是我的 API 客户端代码：@/src/api/client.js
还有端点实现：@/src/server/routes/users.js

我做错了什么？
```

这种方法给 Cline 提供了它需要的一切 - 确切的错误输出、客户端代码和服务器实现 - 而无需我手动复制任何内容。

下次您遇到命令输出或构建错误问题时，尝试使用 `@terminal` 而不是复制输出。您将获得更准确的帮助，因为 Cline 可以看到格式正确的完整终端上下文。

## 幕后工作原理

当您在消息中使用终端提及时，幕后会发生以下情况：

1. 当您发送消息时，Cline 检测文本中的 `@terminal` 模式
2. 扩展调用 `getLatestTerminalOutput()` 来访问 VSCode 的终端 API
3. 它捕获您活动终端的最近输出缓冲区
4. 终端输出以结构化格式附加到您的消息中：

    ```
    <terminal_output>
    $ npm run test
    > project@1.0.0 test
    > jest

    FAIL src/components/__tests__/Button.test.js
    ● Button component › renders correctly

    [格式保持完整的完整终端输出]
    </terminal_output>
    ```

5. 这个包含嵌入终端输出的增强消息被发送给 AI
6. AI 现在可以"看到"格式保持完整的完整终端输出

每当您使用终端提及时，这个过程都会自动发生，给 AI 提供对您命令结果、错误消息和其他终端输出的访问权限，而您无需手动复制它。

## 终端问题故障排除

如果您遇到终端提及或终端集成的一般问题（如"Shell Integration Unavailable"或命令不显示输出），请参考我们的综合[终端集成故障排除指南](/troubleshooting/terminal-integration-guide)。

常见问题包括：

-   终端提及不捕获输出
-   Cline 聊天中显示"Shell Integration Unavailable"消息
-   命令执行但输出对 Cline 不可见
-   终端集成工作不一致

故障排除指南提供了特定平台的解决方案和详细的配置步骤来解决这些问题。
