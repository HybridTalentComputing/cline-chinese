@ 提及是 Cline 最强大的功能之一，让您可以将外部上下文无缝地引入对话中。无需复制和粘贴代码、错误消息或文档，您只需用 @ 符号引用它们即可。

当您在聊天输入中输入 `@` 时，Cline 会显示可用提及类型的菜单。这些提及让您可以直接在对话中引用文件、文件夹、问题、终端输出、git 更改，甚至网络内容。

## 可用的 @ 提及

Cline 支持几种类型的 @ 提及，每种都旨在将不同类型的上下文引入您的对话中：

## 为什么 @ 提及很重要

@ 提及通过以下方式改变您与 Cline 的交互：

1. **消除复制粘贴**：不再需要复制和粘贴代码、错误消息或终端输出。只需直接引用它们。

2. **保持上下文**：Cline 看到完整的上下文，包括导入、相关函数和可能相关的周围代码。

3. **保持格式**：终端输出、错误消息和网络内容保持其格式，使它们更容易理解。

4. **启用复杂工作流程**：组合多个 @ 提及以给 Cline 提供问题的完整图景：

    ```
    我遇到这些错误：@problems

    这是我的组件：@/src/components/Form.jsx
    还有 API 端点：@/src/api/users.js

    错误发生在提交时：@terminal

    我认为这个提交可能导致了它：@a1b2c3d
    ```

## 开始使用

要使用 @ 提及：

1. 在聊天输入中输入 `@`
2. 从菜单中选择提及类型或继续输入
3. 对于文件和文件夹，浏览您的工作区结构
4. 像往常一样发送消息

Cline 将自动处理提及并将引用的内容包含在发送给 AI 的上下文中。

在您与 Cline 的下一次对话中尝试使用 @ 提及 - 当您可以无缝地引入外部上下文时，您会对交互变得多么高效和有效感到惊讶。

## 幕后工作原理

当您在消息中使用 @ 提及时，幕后会发生一个复杂的过程：

1. **检测**：当您发送消息时，Cline 使用正则表达式扫描文本中的 @ 提及模式
2. **处理**：对于每个检测到的提及，Cline：
    - 确定提及类型（文件、文件夹、问题、终端、git、URL）
    - 获取相关内容（文件内容、终端输出等）
    - 适当地格式化内容
3. **增强**：原始消息通过结构化数据增强：

    ```
    您的原始消息与 @/path/to/file

    <file_content path="/path/to/file">
    [完整文件内容]
    </file_content>
    ```

4. **上下文包含**：这个包含所有嵌入内容的增强消息被发送给 AI 模型
5. **无缝响应**：AI 现在可以"看到"所有引用的内容，就像您手动复制和粘贴它们一样

每当您使用 @ 提及时，这整个过程都会自动和无缝地发生，给 AI 提供完整的上下文，而您无需手动复制任何内容。

每种类型的 @ 提及都有其特定的实施细节，您可以在各自的文档页面中找到。
