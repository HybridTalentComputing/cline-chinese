文件提及让您可以将工作区中的任何文件直接拉入与 Cline 的对话中。无需再复制和粘贴代码片段 - 只需输入 `@/` 并指向您需要帮助的文件。

当您在聊天中输入 `@/` 时，Cline 会显示您的工作区文件。浏览文件夹，选择您想要的文件，它立即对 Cline 可用 - 包括所有导入、相关函数和周围上下文。

我在调试时经常使用文件提及。无需尝试弄清楚要复制代码的哪些部分，我只需直接引用文件：

```
当我的表单提交时遇到这个错误：@terminal

这是我的组件：@/src/components/ContactForm.jsx

还有 API 端点：@/src/api/contact.js

我遗漏了什么？
```

这给 Cline 提供了它需要的一切 - 错误消息、组件代码和 API 端点 - 而无需我复制任何内容。Cline 可以看到导入、依赖项和可能导致问题的所有周围上下文。

当您处理跨越多个文件的复杂错误时，文件提及会大放异彩。以前，我必须仔细复制每个相关文件，确保没有遗漏任何重要内容。现在我只需用 `@/` 引用每个文件，Cline 就能获得完整的图景。

下次您遇到问题时，尝试使用文件提及而不是复制代码。您将节省时间并获得更好的答案，因为 Cline 拥有它需要的所有上下文。

## 幕后工作原理

当您在消息中使用文件提及时，幕后会发生以下情况：

1. 当您发送消息时，Cline 检测文本中的 `@/path/to/file` 模式
2. 扩展解析相对于工作区根目录的文件路径
3. 检查文件是二进制（如图像）还是基于文本
4. 对于文本文件，它读取完整的文件内容
5. 文件内容以结构化格式附加到您的消息中：
    ```
    <file_content path="path/to/file">
    [完整文件内容]
    </file_content>
    ```
6. 这个包含嵌入文件内容的增强消息被发送给 AI
7. AI 现在可以"看到"完整的文件内容，就像您复制和粘贴它一样

每当您使用文件提及时，这个无缝过程都会自动发生，给 AI 提供完整的上下文，而您无需手动复制任何内容。
