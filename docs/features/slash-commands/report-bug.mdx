当您遇到 Cline 的奇怪问题时，`/reportbug` 绝对是一个救星。这个命令不是让您记住 GitHub 对错误报告要求的所有细节，而是将 Cline 变成您的个人错误报告助手。

它指导您收集正确错误报告所需的所有信息，然后直接发送到我们的 GitHub issues 页面，包含所有正确的格式和系统详细信息。

#### 使用 `/reportbug` 斜杠命令

当您遇到看起来不对的奇怪事情时：

-   只需在聊天中输入 `/reportbug`
-   Cline 将指导您完成我们需要的所有细节：
    -   描述问题的快速标题
    -   实际发生的情况 vs. 您期望的情况
    -   重现错误的步骤
    -   您看到的任何相关输出或错误
    -   可能帮助我们修复它的额外上下文
-   您将在提交之前审查所有内容
-   一旦您批准，它会打开一个完美格式化的 GitHub issue，包含您的所有信息以及自动系统详细信息

#### 示例

上周我遇到了一个奇怪的错误，Cline 在读取大文件时一直超时。我没有尝试记住所有 GitHub 模板字段，只是输入了 `/reportbug`，Cline 指导我完成了整个过程。

它询问我试图做什么，发生了什么，以及导致问题的确切步骤。最好的部分是它自动包含我的操作系统版本、Cline 版本以及我们开发人员需要的所有技术细节。

几秒钟后，我创建了一个正确格式化的 GitHub issue，而无需自己查找任何信息。
