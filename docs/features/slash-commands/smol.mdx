`/smol`（或其别名 `/compact`）是一个斜杠命令，压缩您的对话历史，同时保留基本上下文。

与创建新任务的 `/newtask` 不同，`/smol` 将您当前的对话压缩为全面的摘要，释放上下文窗口空间，同时允许您在同一任务中继续工作。

将其想象为总结对话的相关部分，同时丢弃其余部分。

#### 使用 `/smol` 斜杠命令

当您的上下文窗口正在填满但您想在同一任务中继续时：

<Frame>
	<img src="https://storage.googleapis.com/cline_public_images/docs/assets/smol.png" alt="使用 /smol 斜杠命令" />
</Frame>

-   在聊天输入字段中输入 `/smol`（或其别名 `/compact`）
-   Cline 将分析您的对话并创建一个保留基本信息的详细摘要
-   您将有机会审查此摘要并在需要时提供反馈
-   一旦接受，详细的对话历史将被此压缩版本替换

#### 示例

当我在复杂的调试会话中深入并需要在同一任务中继续时，我使用 `/smol`。在探索多种方法并检查几个文件后，我的上下文窗口被所有的来回交流填满了。

通过使用 `/smol`，我可以将所有探索压缩为简洁的摘要，捕捉我们学到的内容、我们检查的文件以及我们尝试的方法。这释放了空间来继续调试，而不会失去我们获得的见解。

与 `/newtask` 的关键区别是我保持在相同的对话流程中，而不是创建单独的任务。这在我处于某事中间且不想切换上下文时特别有用。

#### 灵感

以下是我在工作流程中使用 `/smol` 的强大方式：

-   在长时间的头脑风暴会议期间，我使用 `/smol` 在实施选定解决方案之前压缩我们的探索，都在同一任务中。
-   当调试涉及多个文件检查和测试运行的复杂问题时，我使用 `/smol` 总结我们学到的内容，同时继续调试过程。
-   对于迭代开发，我在完成每个功能后使用 `/smol` 压缩实施细节，同时保持关键决策和方法可访问。
-   当从多个来源收集需求时，我在进入设计阶段之前使用 `/smol` 将基本需求提炼为简洁摘要。

#### Smol vs Newtask

人们经常问我什么时候使用 `/smol` vs `/newtask`。坦率地说，这是个人偏好和您试图实现的目标的问题。以下是一些指导原则：

-   当您处于某事中间并想在同一任务中继续时，使用 `/smol`。当您深入调试流程或头脑风暴会议且不想打破您的势头时，这是完美的。缺点？一旦您压缩历史，您就无法恢复那些详细的对话。
-   当您处于逻辑转换点并想重新开始时，使用 `/newtask`。它非常适合从计划转向实施，或者当您想保留完整的对话历史时（因为它创建新任务而不是覆盖您当前的任务）。
