计划与执行模式代表了 Cline 的结构化 AI 开发方法，强调在实施之前进行深思熟虑的规划。这种双模式系统帮助开发者创建更易维护、更准确的代码，同时减少迭代时间。

#### 计划模式：先思考

计划模式是您和 Cline 确定要构建什么以及如何构建的地方。在此模式下，Cline：

-   可以读取您的整个代码库以了解上下文
-   不会对您的文件进行任何更改
-   专注于理解要求和创建策略
-   帮助在您编写一行代码之前识别潜在问题

#### 执行模式：构建它

一旦您有了计划，就切换到执行模式。现在 Cline：

-   拥有所有构建功能
-   可以对您的代码库进行更改
-   仍然记住您规划会话中的所有内容
-   执行您一起制定的策略

### 工作流程指南

当我开发新功能或修复复杂错误时，以下方法对我有效：

1. 我在计划模式中开始，告诉 Cline 我想构建什么
2. Cline 帮助我探索代码库，查看相关文件
3. 我们一起确定最佳方法，考虑边缘情况和潜在问题
4. 当我对我们的计划有信心时，我切换到执行模式
5. Cline 基于我们的规划实施解决方案

-   执行商定的计划
-   对您的代码库进行更改

#### 3. 根据需要迭代

复杂项目通常需要多个计划-执行周期：

-   遇到意外复杂性时返回计划模式
-   使用执行模式实施解决方案
-   在确保质量的同时保持开发势头

### 最佳实践

#### 规划阶段

1. 全面了解要求
2. 提前分享相关上下文
3. 如果 Cline 没有读取相关文件，请指向相关文件
4. 在实施之前验证方法

#### 实施阶段

1. 遵循既定计划
2. 监控目标进度
3. 跟踪更改及其影响
4. 记录重要决定

### 高级用户技巧

#### 增强规划

-   使用计划模式在实施之前探索边缘情况
-   遇到意外复杂性时切换回计划模式
-   利用[文件读取](/features/at-mentions/file-mentions)早期验证假设
-   让 Cline 编写计划的 markdown 文件以供将来参考

### 常见模式

#### 何时使用每种模式

我发现计划模式在以下情况下效果最佳：

-   开始一些方法不明确的新事物
-   调试我不确定问题所在的棘手问题
-   做出将影响代码库多个部分的架构决策
-   尝试理解复杂的工作流程或功能

执行模式非常适合：

-   实施我们已经规划好的解决方案
-   在方法明确的情况下进行常规更改
-   遵循代码库中既定的模式
-   运行测试并进行微调
