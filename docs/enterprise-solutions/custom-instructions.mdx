## 为团队构建自定义指令

**创建标准化的项目指令确保所有团队成员在一致的指导原则下工作。首先记录项目的技术基础，然后确定哪些信息需要包含在指令中。确切的范围将根据团队的需求而变化，但通常最好提供尽可能多的信息。通过创建所有团队成员都遵循的综合指令，您建立了对代码应该如何编写、测试和部署的共同理解，从而产生更可维护和一致的软件。**

---

以下是为团队自定义指令考虑的一些主题和示例：

1. **测试框架和特定命令**
    - "所有组件必须包含 Jest 测试，覆盖率至少为 85%。在提交任何拉取请求之前使用 `npm run test:coverage` 运行测试。"
2. **明确的库偏好**
    - "使用 React Query 进行数据获取和状态管理。除非复杂全局状态特别需要，否则避免使用 Redux。对于样式，使用 Tailwind CSS 和我们在 `src/styles/theme.js` 中找到的自定义主题配置。"
3. **在哪里找到文档**
    - "所有 API 文档都在我们内部的 Notion 工作空间中，位于'工程 > API 参考'下。对于组件使用示例，请参考我们在 `https://storybook.internal.company.com` 的 Storybook 实例"
4. **使用哪些 MCP 服务器，以及用于什么目的**
    - "对于数据库操作，使用 Postgres MCP 服务器，凭据存储在 1Password 的'开发 > 数据库'下。对于部署，使用 AWS MCP 服务器，该服务器需要来自 IAM 的部署角色。请参阅 `docs/mcp-setup.md` 了解配置说明。"
5. **特定于项目的编码约定**
    - "使用 PascalCase 命名所有 React 组件，使用 camelCase 命名所有辅助函数。将组件放在 `src/components` 目录中，按功能组织，而不是按类型组织。始终使用 TypeScript 接口进行属性定义。"
