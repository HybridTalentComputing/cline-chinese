### 概述

Cline 提供使用 AWS 凭据或 AWS 配置文件访问 AWS Bedrock 服务的选项。建议使用 SSO/联合角色而非传统 IAM 配置；本指南描述如何配置您的环境，使 Cline 使用 SSO 角色进行身份验证。

---

### 配置步骤

1. 安装 [最新版本](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) 的 AWS CLI

    - 按照 AWS 文档安装您操作系统特定版本的 AWS CLI

2. 使用 AWS CLI [配置 IAM 身份验证](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html)

    - 如果您还没有通过 IAM Identity Center 访问 AWS，请按照 [IAM 用户指南](https://docs.aws.amazon.com/singlesignon/latest/userguide/getting-started.html) 设置 IAM 用户和角色。确保您有 `PowerUserAccess` 角色。
    - 如果您通过雇主访问 AWS，请打开您的 AWS 访问门户并找到适当的账户。确保您有 `PowerUserAccess` 权限。
    - 打开 `Access keys` 链接并记下 `SSO start URL` 和 `SSO region`，这些在下一步中需要

3. 继续使用 [the `aws configure sso` CLI 向导](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html#cli-configure-sso-configure) 配置您的配置文件

    - 配置完成后，使用以下命令对 AWS CLI 进行身份验证：`aws sso login --profile <AWS-profile-name>`
    - 记下您附加到 AWS 账户的配置文件名称，这在以下步骤中配置 Cline 时需要

4. 如果您还没有这样做，请安装 VSCode 和 Cline 扩展。请查阅 [开始使用](/getting-started) 页面获取指导。

5. 打开 Cline 扩展，然后点击设置按钮 ⚙️ 选择您的 API 提供商。
    - 从 API 提供商下拉菜单中，选择 AWS Bedrock
    - 选择 AWS Profile 单选按钮，然后输入步骤 3 中的 AWS Profile Name
    - 从下拉菜单中选择您的 AWS 区域
    - 对于某些模型，需要选择跨区域推理复选框
